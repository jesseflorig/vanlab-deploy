# Implementation Plan: Prometheus Longhorn Monitoring

**Branch**: `009-prometheus-longhorn-monitoring` | **Date**: 2026-04-01 | **Spec**: [spec.md](spec.md)
**Input**: "Deploy kube-prometheus-stack, Longhorn ServiceMonitor, and a community longhorn dashboard"

## Summary

Deploy kube-prometheus-stack via a new `kube-prometheus-stack` Ansible role, enable the
Longhorn ServiceMonitor by adding `monitoring.enabled: true` to the Longhorn Helm values,
and provision the community Longhorn dashboard (ID 13032) automatically via Grafana's
built-in dashboard sidecar. All storage backed by Longhorn PVCs.

## Technical Context

**Language/Version**: YAML (Ansible 2.x) — existing project conventions
**Primary Dependencies**: `prometheus-community/kube-prometheus-stack` Helm chart; Longhorn (already deployed)
**Storage**: Longhorn `storageClassName: longhorn` — Prometheus 20Gi, Grafana 5Gi, Alertmanager 5Gi
**Testing**: Manual — `kubectl get pods -n monitoring`, Grafana UI, Prometheus targets page
**Target Platform**: K3s arm64 (Raspberry Pi CM5)
**Project Type**: Infrastructure automation (Ansible role + Helm)
**Performance Goals**: Grafana loads within 3s; metrics visible within 5 min of deploy
**Constraints**: Idempotent (Principle II); K3s control-plane scrapers must be disabled
**Scale/Scope**: 4-node homelab cluster; ~5000 metric series; ~2-week retention
