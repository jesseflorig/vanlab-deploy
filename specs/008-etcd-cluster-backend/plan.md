# Implementation Plan: etcd Cluster Backend

**Branch**: `008-etcd-cluster-backend` | **Date**: 2026-04-01 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/008-etcd-cluster-backend/spec.md`

## Summary

Migrate the K3s cluster datastore from embedded SQLite to embedded etcd by updating
`k3s-deploy.yml` to pass `--cluster-init` on the first server node and support additional
server nodes joining the etcd quorum. A clean rebuild of the live cluster is required.
No new roles or tools are introduced — this is a targeted change to a single playbook.

## Technical Context

**Language/Version**: YAML (Ansible 2.x) — existing project conventions
**Primary Dependencies**: K3s (already installed), embedded etcd (bundled with K3s — no separate install)
**Storage**: N/A — etcd state stored at `/var/lib/rancher/k3s/server/db/` on server nodes
**Testing**: Manual — run playbook, verify `kubectl get nodes`, verify ArgoCD sync
**Target Platform**: Raspberry Pi OS arm64 (existing nodes)
**Project Type**: Infrastructure automation (Ansible playbook)
**Performance Goals**: Node join within 5 minutes; full rebuild within 20 minutes
**Constraints**: Idempotent (Principle II); clean rebuild required (no in-place migration)
**Scale/Scope**: 1–6 nodes; designed for 1 server now, expandable to 3 without changes
