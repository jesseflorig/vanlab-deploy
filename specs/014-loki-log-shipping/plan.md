# Implementation Plan: Add Grafana Loki Log Aggregation

**Branch**: `014-loki-log-shipping` | **Date**: 2026-04-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/014-loki-log-shipping/spec.md`

## Summary

Deploy Grafana Loki (SingleBinary mode) and Grafana Alloy (DaemonSet log shipper) as Ansible-managed Helm roles. Loki stores logs on a Longhorn PVC. Alloy ships pod logs and journald system logs from all 6 nodes. The existing Grafana instance (kube-prometheus-stack) is updated to include Loki as an additional datasource. No new UI is deployed.

## Technical Context

**Language/Version**: YAML (Ansible 2.x) — follows existing project conventions
**Primary Dependencies**:
- `grafana/loki` Helm chart v6.55.0 — log aggregation store (SingleBinary mode)

- `prometheus-community/kube-prometheus-stack` (already deployed) — Grafana datasource update
**Storage**: Longhorn `storageClass: longhorn` — Loki 20Gi PVC
**Target Platform**: K3s on arm64 Raspberry Pi CM5 (Debian-based)
**Project Type**: Infrastructure Ansible role + Helm chart
**Constraints**: Must follow all 11 Constitution principles; Ansible-managed (not ArgoCD)

## Key Implementation Details

### roles/loki/defaults/main.yml

```yaml
loki_namespace: monitoring
loki_chart_version: "6.55.0"
loki_storage_size: "20Gi"
loki_retention_period: "168h"   # 7 days
```

### roles/loki/templates/values.yaml.j2 (structure)

```yaml
deploymentMode: SingleBinary

loki:
  auth_enabled: false
  limits_config:
    retention_period: {{ loki_retention_period }}
  storage:
    type: filesystem
  storageConfig:
    filesystem:
      directory: /var/loki/chunks
  compactor:
    retention_enabled: true
    delete_request_store: filesystem
    working_directory: /var/loki/retention

singleBinary:
  replicas: 1
  persistence:
    enabled: true
    size: {{ loki_storage_size }}
    storageClass: longhorn

# Disable distributed-mode components
distributor:   { replicas: 0 }
ingester:      { replicas: 0 }
querier:       { replicas: 0 }
queryFrontend: { replicas: 0 }

# No external ingress — accessed internally by Alloy and Grafana
gateway:
  enabled: false
```

### roles/alloy/templates/config.alloy.j2 (structure)

```hcl
// Discover all pods in the cluster
discovery.kubernetes "pods" {
  role = "pod"
}

// Collect pod logs
loki.source.kubernetes "pods" {
  targets    = discovery.kubernetes.pods.targets
  forward_to = [loki.write.default.receiver]
}

// Collect journald system logs
loki.source.journal "system" {
  forward_to = [loki.write.default.receiver]
  labels     = { job = "systemd-journal" }
}

// Push to Loki
loki.write "default" {
  endpoint {
    url = "http://loki.{{ loki_namespace }}.svc.cluster.local:3100/loki/api/v1/push"
  }
}
```

### kube-prometheus-stack values.yaml.j2 addition

```yaml
grafana:
  additionalDataSources:
    - name: Loki
      type: loki
      url: http://loki.monitoring.svc.cluster.local:3100
      access: proxy
      isDefault: false
      jsonData:
        maxLines: 1000
```

### roles/alloy Helm values (key settings)

- `alloy.configMap.create: true` with rendered `config.alloy.j2`
- `controller.type: daemonset` — one Alloy per node
- Tolerations for control-plane nodes (so all 3 servers also ship logs)
- Host path mounts: `/var/log/pods`, `/run/log/journal`, `/etc/machine-id`
- `alloy.mounts.varlog: true` — enable host log mount

### run_once pattern

All Helm tasks in `roles/loki/tasks/main.yml` and `roles/alloy/tasks/main.yml` MUST use `run_once: true` (same pattern as all other roles, to avoid Helm race on 3-server cluster).
