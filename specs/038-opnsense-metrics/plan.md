# Implementation Plan: OPNsense Metrics Collection

**Branch**: `038-opnsense-metrics` | **Date**: 2026-04-04 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/038-opnsense-metrics/spec.md`

## Summary

Deploy the AthennaMind opnsense-exporter (v0.0.14) as a stateless ArgoCD-managed workload in the `monitoring` namespace. The exporter calls the OPNsense REST API at `10.1.1.1` using an API key stored as a SealedSecret, and exposes Prometheus metrics on port 8080. A `ScrapeConfig` CRD resource tells the existing Prometheus Operator to scrape it. A Grafana dashboard (gnetId 21113) is provisioned via the kube-prometheus-stack Helm values.

## Implementation Notes

### OPNsense API User Setup (manual prerequisite)
Before the playbook can generate the SealedSecret, an API key must be created manually in OPNsense WebUI (System → Access → Users) with the following ACL grants:
- Diagnostics: ARP Table, Firewall statistics, Netstat
- Status: Services, System Status, Gateways
- VPN: WireGuard, OpenVPN Instances
- System: Firmware, Cron, Unbound DNS

Enable Unbound extended statistics: Services → Unbound DNS → Advanced → Extended statistics: ✓

Once generated, the one-time key+secret values are stored in `group_vars/all.yml`:
```yaml
opnsense_api_key: "<key>"
opnsense_api_secret: "<secret>"
```

### SealedSecret Generation
```bash
ansible-playbook playbooks/utilities/seal-secrets.yml --tags opnsense-exporter
```
This generates `manifests/monitoring/prereqs/sealed-secrets.yaml` — safe to commit.

### Firewall Rule
OPNsense must permit `10.1.20.0/24 → 10.1.1.1:443` (HTTPS) for the exporter pod to reach the REST API. Codify via Ansible `community.opnsense` firewall rule task or verify the existing LAN→firewall anti-lockout rule already covers this.

### Grafana Dashboard
Add to `roles/kube-prometheus-stack/templates/values.yaml.j2`:
```yaml
grafana:
  dashboards:
    default:
      opnsense-exporter:
        gnetId: 21113
        revision: 1
        datasource: Prometheus
```
Re-run `ansible-playbook -i hosts.ini playbooks/cluster/services-deploy.yml --tags monitoring` to apply.

### ArgoCD Registration
Add to `group_vars/all.yml` → `argocd_apps`:
```yaml
- name: monitoring-prereqs
  repo: gitadmin/vanlab.git
  path: manifests/monitoring/prereqs
  namespace: monitoring
  revision: main
- name: monitoring-apps
  repo: gitadmin/vanlab.git
  path: manifests/monitoring/exporter
  namespace: monitoring
  revision: main
```
Re-run `--tags argocd-bootstrap` to register the Applications.
